Information about cookies and similar technologies

This page explains, in clear language, how the website uses cookies, sessionStorage, localStorage and similar tools to run the service, manage transfer request forms and measure basic website activity.

What cookies are

Cookies are small text files that a website can store in a user's browser. They are commonly used to keep technical features working, remember preferences or collect usage statistics. A website may also use similar technologies such as sessionStorage or localStorage to save information on the device temporarily or for a longer period.

Types of cookies and tools used on this website

Technical cookies or necessary tools. These are used to make the website and its core features work properly, including request forms and basic technical security.

Basic analytics tools. The website records limited events such as CTA clicks, form opens, form submissions, WhatsApp clicks, UTM parameters and essential page or device information.

Third-party services. The website includes external services or resources such as WhatsApp for direct contact, EmailJS for request delivery from static pages, Google Fonts for web fonts and jsDelivr for loading libraries. These providers may process technical connection data under their own policies.

Technologies actually used on the site

sessionStorage / localStorage. The site temporarily stores selected information to improve the browsing and form experience, such as form drafts, UTM parameters, a technical session identifier and form-open state.

Basic event tracking. The site logs limited events such as button clicks, form submissions, form validation errors, WhatsApp interactions and essential page usage information.

Possible session cookies. If the site is served through components that require a technical session, strictly necessary session cookies may be present for operational reasons.

Why these tools are used

These tools are used to:

1. run the website and transfer request forms correctly;

2. improve form completion and reduce accidental data loss or user errors;

3. manage requests sent through forms or WhatsApp contact;

4. understand, at a basic level, how visitors use the website and which actions they complete.

Consent

Strictly necessary cookies and similar technical tools may be used without prior consent where allowed by applicable law. For non-essential tools, the correct approach is to collect consent before activation.

Users can manage or withdraw their preferences through browser settings and, where available, through a cookie banner or preference center.

How to disable cookies

Users can limit or block cookies from their browser settings. In most browsers it is possible to:

1. see which cookies are stored;

2. delete cookies already saved;

3. block all cookies or only third-party cookies;

4. ask the browser to request confirmation before storing them.

Instructions vary depending on the browser used, such as Chrome, Safari, Firefox, Edge or a mobile browser.

User rights

Under the GDPR, users may have the right to access their data, request rectification, erasure, restriction of processing, object to processing and, where applicable, request data portability. Users may also lodge a complaint with the competent supervisory authority.
